## 1. API 문서 지원

**목표:** 백엔드 API 문서화 도구 추가 및 Ingress를 통한 접근 설정

**작업 내용:**
- SpringDoc OpenAPI (Swagger UI) 설정
  - 의존성 추가
  - `application.properties` 설정
  - API 엔드포인트 문서화
- Ingress 설정에 `/api-docs` 경로 추가
  - `values.yaml`에 path 추가
  - Swagger UI 접근 경로 설정

**예상 결과:**
- `http://message.rahoon.site/api-docs` 또는 `/swagger-ui.html` 접근 가능
- API 엔드포인트 목록 및 테스트 인터페이스 제공

## 2. 간단한 프론트엔드 생성 및 배포

**목표:** 기본 프론트엔드 애플리케이션 생성 및 Kubernetes 배포

**작업 내용:**
- 프론트엔드 프로젝트 생성
  - React 또는 Vue.js 선택
  - 기본 구조 설정
- Docker 이미지 빌드
  - Dockerfile 작성
  - 빌드 스크립트에 프론트엔드 추가
- Helm 차트에 프론트엔드 서비스 추가
  - `values.yaml`에 프론트엔드 설정 추가
  - Ingress에 `/` 경로로 프론트엔드 라우팅
- 배포 및 테스트
  - 이미지 빌드 및 푸시
  - Helm 차트 배포
  - 브라우저에서 접근 확인

**예상 결과:**
- `http://message.rahoon.site/` 접근 시 프론트엔드 화면 표시
- `/api` 경로로 백엔드 API 호출 가능

## 참고사항

- 프론트엔드는 정적 파일 서빙 (Nginx 기반 이미지 사용 가능)
- API 호출 시 CORS 설정 필요할 수 있음
- Ingress 설정에서 프론트엔드와 백엔드 경로 분리 확인
